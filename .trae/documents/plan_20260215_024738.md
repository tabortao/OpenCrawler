## 开发计划：多平台网页 Markdown 提取 API

### 第一步：创建项目规则文件
- 创建 `.trae/rules/project_rules.md`，包含项目技术栈、目录结构、编码规范等

### 第二步：创建项目基础结构
- 创建 `requirements.txt`：FastAPI, crawlee, playwright, trafilatura, python-dotenv, httpx, uvicorn
- 创建 `.env.example`：环境变量配置模版

### 第三步：实现核心模块
- **utils.py**：Cookie 解析工具、URL 验证、文本处理函数
- **crawler.py**：基于 Crawlee + Playwright 的网页抓取逻辑，支持：
  - 动态渲染与页面滚动
  - 浏览器指纹伪装
  - 平台特定选择器等待
  - Cookie 与代理注入
- **main.py**：FastAPI 应用入口，包含 `/extract` 接口与异常处理

### 第四步：验证与测试
- 检查代码语法与类型
- 确保所有模块正确导入
