# 通用插件优化计划

## 任务目标

对通用插件的文章爬取能力进行全面优化与增强，使其达到与微信公众号插件、少数派插件同等水平的文章爬取效果。

## 优化方向分析

### 当前通用插件与专项插件对比

| 功能 | 通用插件 | 微信公众号 | 少数派 |
|------|----------|------------|--------|
| 内容选择器 | 基础选择器列表 | 专用选择器 `.rich_media_content` | 专用选择器 `article` |
| 内容清理 | 无 | 丰富的清理逻辑 | 基础清理 |
| 图片提取 | 基础提取 | `data-original` 优先提取 | `data-original` 优先提取 |
| 标题处理 | 基础提取 | 基础提取 | 去除站点后缀 |
| 代码块处理 | 基础转换 | 基础转换 | 基础转换 |
| 表格处理 | 基础转换 | 基础转换 | 基础转换 |

### 主要差距

1. **内容选择器不足**: 通用插件的选择器覆盖面不够广
2. **缺少内容清理**: 没有类似微信公众号的内容清理逻辑
3. **缺少内容后处理**: 没有对提取的内容进行清理和优化
4. **图片提取不够智能**: 没有优先使用高清图（data-original）
5. **动态加载处理不足**: 没有针对懒加载等特殊情况的处理

## 优化任务清单

### 1. 增强内容选择器覆盖

- [ ] 扩展 `_extract_main_content` 方法的选择器列表
- [ ] 添加更多常见网站的内容选择器
- [ ] 实现基于页面结构的智能选择器优先级判断

### 2. 添加内容清理功能

- [ ] 提取通用内容清理方法到 BaseCrawler
- [ ] 移除导航栏、侧边栏、广告等无关内容
- [ ] 清理样式残留和空元素

### 3. 增强图片提取能力

- [ ] 优先提取 `data-original` 属性（高清图）
- [ ] 处理懒加载图片（data-src）
- [ ] 过滤头像、图标等小图
- [ ] 添加图片 URL 规范化处理

### 4. 优化标题提取

- [ ] 添加多种标题提取策略
- [ ] 处理特殊站点标题格式（如去除站点后缀）
- [ ] 处理 Unicode 编码和转义字符

### 5. 增强特殊内容处理

- [ ] 优化代码块转换（保留语言标识）
- [ ] 改进表格转换（处理复杂表格）
- [ ] 处理嵌入内容（视频、iframe 等）

### 6. 优化动态加载处理

- [ ] 添加更智能的滚动加载策略
- [ ] 等待懒加载内容完成
- [ ] 处理无限滚动页面

## 实现步骤

### 第一阶段：基础优化（选择器 + 清理）

1. 扩展内容选择器列表
2. 添加通用内容清理方法

### 第二阶段：内容质量提升（图片 + 标题）

3. 改进图片提取逻辑
4. 优化标题提取策略

### 第三阶段：高级特性（特殊内容 + 动态加载）

5. 增强特殊内容处理
6. 优化动态加载处理

## 预期效果

- 通用插件对各类网站的文章爬取成功率提升
- 提取的内容更加干净、结构完整
- 图片提取更加完整（高清优先）
- 代码块、表格等特殊内容保留更好
