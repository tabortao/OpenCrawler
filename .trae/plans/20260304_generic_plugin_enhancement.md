# 通用插件文章爬取能力优化计划

## 任务目标
提升通用插件的文章爬取能力，使其达到与微信公众号插件、少数派插件同等水平的爬取效果。

## 现状分析

### 微信公众号插件特点
1. 精确的内容选择器：`.rich_media_content`, `#js_content`
2. 针对性的内容清理：过滤视频播放器、分享按钮等微信特有元素
3. 图片处理：`data-src` 属性提取
4. 样式标题转换：将微信样式的加粗标题转换为标准 Markdown 标题

### 少数派插件特点
1. 精确的内容选择器：`article`, `.article-content`, `.content`
2. 标题清理：移除" - 少数派"后缀
3. 图片处理：`data-original` 高清图片提取
4. 过滤头像和缩略图

### 通用插件当前问题
1. 内容选择器列表不够智能，可能无法准确识别主要内容区域
2. 没有使用 trafilatura 等专业的内容提取库
3. 标题提取和清理不够完善
4. 动态内容处理不够充分
5. 图片提取策略可以优化
6. 没有内容质量评估机制
7. 特殊格式处理不够完善

## 优化方案

### 1. 引入智能内容提取 (优先级: 高)
- 集成 trafilatura 库进行智能内容识别
- 作为内容提取的兜底方案
- 与现有选择器策略结合使用

### 2. 增强内容选择器策略 (优先级: 高)
- 扩展内容选择器列表，覆盖更多网站结构
- 添加基于内容密度的选择器评分机制
- 支持基于语义的内容区域识别

### 3. 优化标题提取 (优先级: 高)
- 扩展标题提取选择器
- 增强标题清理规则
- 支持 Open Graph 和 Schema.org 元数据提取

### 4. 增强动态内容处理 (优先级: 中)
- 改进懒加载图片等待机制
- 添加动态内容加载检测
- 支持 SPA 页面的内容等待

### 5. 改进图片提取 (优先级: 中)
- 支持更多懒加载属性格式
- 添加图片质量过滤
- 支持响应式图片(srcset)提取

### 6. 内容质量评估与后处理 (优先级: 中)
- 添加内容质量评分机制
- 智能去除无关内容
- 优化 Markdown 输出格式

### 7. 特殊格式处理增强 (优先级: 中)
- 改进代码块识别和格式化
- 优化表格处理
- 支持数学公式提取
