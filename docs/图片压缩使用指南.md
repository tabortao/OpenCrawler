# 图片压缩功能使用指南

OpenCrawler 支持在下载图片时自动压缩，支持 JPEG、PNG、WebP 格式，可有效节省存储空间。

## 功能特点

- 支持 JPEG、PNG、WebP 格式压缩
- 可自定义压缩质量 (1-95)
- 返回压缩统计信息（原始大小、压缩后大小、节省比例）
- API 和 MCP 都支持

---

## curl API 使用

### 基本用法

```bash
# 下载图片但不压缩
curl -X POST http://127.0.0.1:8000/api/v1/articles \
  -H "Content-Type: application/json" \
  -d '{"url": "https://sspai.com/post/105218", "download_images": true}'
```

### 启用压缩

```bash
# 下载并压缩图片（默认质量 85）
curl -X POST http://127.0.0.1:8000/api/v1/articles \
  -H "Content-Type: application/json" \
  -d '{"url": "https://sspai.com/post/105218", "download_images": true, "compress_images": true}'
```

### 指定压缩质量

```bash
# 压缩质量 70（压缩率更高，质量稍低）
curl -X POST http://127.0.0.1:8000/api/v1/articles \
  -H "Content-Type: application/json" \
  -d '{"url": "https://sspai.com/post/105218", "download_images": true, "compress_images": true, "compress_quality": 70}'

# 压缩质量 90（质量更高，压缩率较低）
curl -X POST http://127.0.0.1:8000/api/v1/articles \
  -H "Content-Type: application/json" \
  -d '{"url": "https://sspai.com/post/105218", "download_images": true, "compress_images": true, "compress_quality": 90}'
```

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `url` | string | 必填 | 要提取的网页 URL |
| `download_images` | boolean | false | 是否下载图片到本地 |
| `compress_images` | boolean | false | 是否压缩下载的图片（需先启用 download_images） |
| `compress_quality` | integer | 85 | 压缩质量 (1-95)，值越小压缩率越高 |

### 响应示例

```json
{
  "title": "文章标题",
  "url": "https://sspai.com/post/105218",
  "filepath": "output/2026-02-27_文章标题.md",
  "compress_stats": {
    "total_original": 1500000,
    "total_compressed": 450000,
    "count": 5,
    "saved_bytes": 1050000,
    "saved_percent": 70.0
  }
}
```

### compress_stats 字段说明

| 字段 | 说明 |
|------|------|
| `total_original` | 原始图片总大小（字节） |
| `total_compressed` | 压缩后图片总大小（字节） |
| `count` | 压缩的图片数量 |
| `saved_bytes` | 节省的字节数 |
| `saved_percent` | 节省的百分比 |

## MCP 使用

### 启动 MCP 服务器

```bash
# HTTP 模式（推荐 Trae、Cursor）
micromamba run -p ./venv python mcp_server.py http --port 8765

# stdio 模式（推荐 Cherry Studio）
micromamba run -p ./venv python mcp_server.py stdio
```

### 调用 save_article 工具

在 MCP 客户端中调用 `save_article` 工具，支持以下参数：

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `url` | string | 是 | 网页 URL |
| `download_images` | boolean | 否 | 是否下载图片 |
| `compress_images` | boolean | 否 | 是否压缩图片（需启用 download_images） |
| `compress_quality` | integer | 否 | 压缩质量 (1-95)，默认 85 |

### 示例调用

在 Cherry Studio 或 Trae 中：

```
请帮我保存这篇文章：https://sspai.com/post/105218
需要下载图片并压缩，压缩质量设为 70
```

MCP 会自动调用：

```json
{
  "url": "https://sspai.com/post/105218",
  "download_images": true,
  "compress_images": true,
  "compress_quality": 70
}
```

## 压缩质量建议

| 质量值 | 适用场景 | 特点 |
|--------|----------|------|
| 60-70 | 网页缩略图、大量图片存储 | 压缩率高，质量可接受 |
| 75-85 | 一般文章配图 | 平衡质量和大小 |
| 85-95 | 高质量图片展示 | 质量高，压缩率较低 |

---

## 支持的图片格式

| 格式 | 压缩方式 |
|------|----------|
| JPEG (.jpg, .jpeg) | 有损压缩，使用 quality 参数 |
| PNG (.png) | 无损优化 |
| WebP (.webp) | 有损压缩，使用 quality 参数 |

---

## 注意事项

1. **压缩需要先下载图片**：`compress_images` 参数仅在 `download_images: true` 时生效
2. **PNG 格式**：使用无损优化，quality 参数对 PNG 无效
3. **透明度**：带有透明通道的 PNG/WebP 会自动转换为 RGB 格式
4. **依赖**：需要安装 Pillow 库（已在 requirements.txt 中）
