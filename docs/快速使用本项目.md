## API 认证说明

如果配置了 `API_TOKEN` 环境变量，所有 API 请求都需要携带认证令牌。

### 生成令牌

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

将生成的令牌配置到 `.env` 文件中：

```env
API_TOKEN=your_generated_token_here
```

### 使用令牌

支持两种认证方式：

```bash
# 方式1: Bearer Token（推荐）
curl -H "Authorization: Bearer YOUR_TOKEN" http://127.0.0.1:8000/api/v1/articles

# 方式2: X-API-Token
curl -H "X-API-Token: YOUR_TOKEN" http://127.0.0.1:8000/api/v1/articles
```

### 安全建议

1. 使用至少 32 字节的随机字符串
2. 定期轮换令牌（建议每 90 天）
3. 确保 `.env` 文件不在版本控制中
4. 设置适当的文件权限（Linux/Mac: `chmod 600 .env`）

## 使用CLI

以下示例假设已配置 `API_TOKEN`，请将 `YOUR_TOKEN` 替换为实际令牌。如果未配置令牌，则不需要添加认证头。

```bash
# 提取微信公众号文章（下载图片）
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "https://mp.weixin.qq.com/s/e4s2x2pqfKTlaQ4M9r3DaA", "download_images": true}'

# 提取知乎文章（下载并压缩图片）
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "https://zhuanlan.zhihu.com/p/1946593133073764815", "download_images": true, "compress_images": true}'

# 提取少数派文章（下载并压缩图片，指定压缩质量）
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "https://sspai.com/post/105218", "download_images": true, "compress_images": true, "compress_quality": 70}'

# 提取 GitHub 仓库文章
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "https://github.com/tabortao/gocron", "download_images": true}'

# 提取小红书文章
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "http://xhslink.com/o/50GU2ftTs5j", "download_images": true}'

# 提取阮一峰博客文章
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "https://www.ruanyifeng.com/blog/2024/03/weekly-issue-293.html"}'

# 提取 cnBeta 科技新闻
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "https://www.cnbeta.com.tw/articles/tech/1428967.htm"}'

# 提取酷壳博客文章
curl -X POST http://127.0.0.1:8000/api/v1/articles -H "Content-Type: application/json" -H "Authorization: Bearer YOUR_TOKEN" -d '{"url": "https://coolshell.cn/articles/22422.html"}'
```

### 参数说明

| 参数               | 类型    | 默认值 | 说明                                             |
| ------------------ | ------- | ------ | ------------------------------------------------ |
| `url`              | string  | 必填   | 要提取的网页 URL                                 |
| `download_images`  | boolean | false  | 是否下载图片到本地                               |
| `compress_images`  | boolean | false  | 是否压缩下载的图片（需要先启用 download_images） |
| `compress_quality` | integer | 85     | 压缩质量 (1-95)，值越小压缩率越高，质量越低      |

## 使用MCP服务器

详见[MCP服务器使用说明](mcp_server_usage.md)

MCP 服务器通过 stdio 运行，不需要额外的 API 认证。

### 启动MCP服务器

推荐使用HTTP模式，然后在Cherry Studio中配置MCP服务器就可以使用了。

```bash
# stdio 模式（推荐 Cherry Studio）
micromamba run -p ./venv python mcp_server.py stdio

# SSE 模式（推荐 Cherry Studio HTTP 方式），注意sse和http启动命令是不同的
micromamba run -p ./venv python mcp_server.py sse --port 8765

# HTTP 模式（推荐 Trae、Cursor），注意sse和http启动命令是不同的
micromamba run -p ./venv python mcp_server.py http --port 8765
```

## 特殊文件夹说明

### storage 文件夹

这是 **Crawlee** (Playwright 爬虫框架) 自动生成的存储目录，包含：

| 子目录              | 作用                                         |
| ------------------- | -------------------------------------------- |
| `key_value_stores/` | 键值存储，保存爬虫状态、统计数据、会话信息等 |
| `request_queues/`   | 请求队列，管理待爬取的 URL                   |

## browser_data 文件夹

这是 **浏览器数据目录**，用于存储：

- Cookie 和登录状态
- 浏览器缓存
- 用户会话数据
