# 产品需求文档 (PRD): 多平台网页 Markdown 提取 API

## 1. 项目概述

构建一个基于 FastAPI 的高性能 Web API，能够接收 URL（特别针对 GitHub、微信公众号、知乎、小红书、少数派等平台），利用 Playwright 模拟真实浏览器行为，绕过反爬机制，提取网页正文并转换为干净的 Markdown 格式。采用插件化架构设计，易于扩展和维护。

## 2. 技术栈

| 组件 | 技术选型 |
|------|----------|
| 框架 | FastAPI (Python 3.10+) |
| 浏览器自动化 | Playwright (Chromium) |
| 内容解析 | Trafilatura |
| 配置管理 | python-dotenv |
| 异步处理 | asyncio |

## 3. 核心功能

### 3.1 增强型网页抓取

- **动态渲染**: 支持 JavaScript 渲染，处理 GitHub README、知乎评论和小红书笔记
- **模拟行为**: 自动执行页面滚动触发懒加载内容
- **反爬避让**: 浏览器指纹伪装，防止被识别为无头浏览器

### 3.2 平台深度优化

| 平台 | 选择器 | 特殊处理 |
|------|--------|----------|
| GitHub | `.markdown-body` | 无 |
| 知乎 | `.Post-RichText` | Cookie 注入、标题过滤私信、内容过滤导航 |
| 小红书 | `.note-content` | Cookie + 代理 |
| 微信公众号 | `.rich_media_content` | data-src 图片提取 |
| 少数派 | `article` | data-original 高清图片 |
| 通用网站 | `article, main` | 自动内容识别 |

### 3.3 图片处理

- 自动提取微信文章中的 `data-src` 图片
- 图片下载到本地 `images/` 目录
- Markdown 中使用相对路径引用

### 3.4 接口规范

#### GET /api/v1/health
健康检查接口

#### GET /api/v1/pages/title?url=<URL>
获取网页标题

#### POST /api/v1/pages/extract
提取页面内容返回 JSON

#### POST /api/v1/articles
提取并保存为 Markdown 文件

**成功响应:**
```json
{
  "status": "success",
  "title": "文章标题",
  "url": "原始URL",
  "filepath": "output/2026-02-24_标题.md"
}
```

## 7. 错误处理

| 状态码 | 错误类型 | 说明 |
|--------|----------|------|
| 400 | INVALID_URL | URL 格式不正确 |
| 422 | EMPTY_CONTENT | 页面内容为空或无法解析 |
| 500 | TIMEOUT/CRAWL_ERROR | 抓取超时或代理失效 |
