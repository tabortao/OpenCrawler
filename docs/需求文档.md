# 产品需求文档 (PRD): 多平台网页 Markdown 提取 API

## 1. 项目概述

构建一个基于 FastAPI 的高性能 Web API，能够接收 URL（特别针对 GitHub、微信公众号、知乎、小红书），利用 Crawlee (Python) 和 Playwright 模拟真实浏览器行为，绕过反爬机制，提取网页正文并转换为干净的 Markdown 格式。

## 2. 技术栈

| 组件 | 技术选型 |
|------|----------|
| 框架 | FastAPI (Python 3.10+) |
| 爬虫引擎 | Crawlee (Python 1.4+) + Playwright (Chromium) |
| 内容解析 | Trafilatura |
| 配置管理 | python-dotenv |
| 异步处理 | asyncio & httpx |

## 3. 核心功能

### 3.1 增强型网页抓取

- **动态渲染**: 支持 JavaScript 渲染，处理 GitHub README、知乎评论和小红书笔记
- **模拟行为**: 自动执行页面滚动触发懒加载内容
- **反爬避让**: Crawlee 浏览器指纹伪装，防止被识别为无头浏览器

### 3.2 平台深度优化

| 平台 | 选择器 | 特殊处理 |
|------|--------|----------|
| GitHub | `.markdown-body` | 无 |
| 知乎 | `.Post-RichText` | Cookie 注入、弹窗关闭 |
| 小红书 | `.note-content` | Cookie + 代理 |
| 微信公众号 | `.rich_media_content` | data-src 图片提取、广告过滤 |

### 3.3 图片处理

- 自动提取微信文章中的 `data-src` 图片
- 图片下载到本地 `images/` 目录
- Markdown 中使用相对路径引用

### 3.4 接口规范

#### GET /extract
提取网页内容返回 JSON

#### GET /save
提取并保存为 Markdown 文件

**成功响应:**
```json
{
  "status": "success",
  "title": "文章标题",
  "url": "原始URL",
  "filepath": "output/2026-02-24_标题.md"
}
```

```env

# 输出配置

# 反爬配置

# 平台 Cookie
ZHIHU_COOKIE=name=value; name2=value2

```

## 7. 错误处理

| 状态码 | 错误类型 | 说明 |
|--------|----------|------|
| 400 | INVALID_URL | URL 格式不正确 |
| 422 | EMPTY_CONTENT | 页面内容为空或无法解析 |
| 500 | TIMEOUT/CRAWL_ERROR | 抓取超时或代理失效 |
