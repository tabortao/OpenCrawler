产品需求文档 (PRD):
多平台网页 Markdown 提取 API
1. 项目概述
构建一个基于 FastAPI 的高性能 Web API。该工具能够接收 URL（特别针对 GitHub、微信公众号、知乎、小红书），利用 Crawlee (Python) 和 Playwright 模拟真实浏览器行为，绕过反爬机制，提取网页正文并转换为干净的 Markdown 格式。
2. 技术栈要求
 * 框架: FastAPI (Python 3.10+)
 * 爬虫引擎: Crawlee (Python 版) + Playwright (Chromium)
 * 内容解析: Trafilatura (用于正文提取与 Markdown 转换)
 * 配置管理: python-dotenv
 * 异步处理: asyncio & httpx
3. 核心功能需求
3.1 增强型网页抓取
 * 动态渲染: 必须支持 JavaScript 渲染，以处理 GitHub README、知乎评论和小红书笔记。
 * 模拟行为: 自动执行页面滚动（ScrollDown）以触发懒加载（Lazy Load）内容。
 * 反爬避让: 启用 Crawlee 的浏览器指纹（Browser Fingerprints）伪装功能，防止被识别为无头浏览器。
3.2 平台深度优化
 * GitHub: 必须等待 .markdown-body 选择器加载完成。
 * 知乎 & 小红书: 支持从环境变量动态注入 Cookie，并支持通过代理服务器（Proxy）请求。
 * 微信公众号: 自动过滤非正文元素（如：关注按钮、底部广告）。
3.3 接口规范 (GET /extract)
 * 输入参数: url (string, required)
 * 输出格式:
   {
  "status": "success",
  "title": "文章标题",
  "url": "原始URL",
  "markdown": "# 转换后的内容..."
}

4. 环境变量与配置 (.env)
项目必须支持通过 .env 文件进行无侵入式配置。AI IDE 应生成如下结构的配置模版：
# Server Configuration
HOST=127.0.0.1
PORT=8000

# Anti-Bot Configuration
PROXY_URL=               # 格式: http://user:pass@host:port
ZHIHU_COOKIE=            # 知乎原始 Cookie 字符串
XHS_COOKIE=              # 小红书原始 Cookie 字符串

5. 逻辑细节与防御性编程
 * Cookie 自动解析: 编写辅助函数，将 .env 中的 name=value; 格式字符串转换为 Playwright 兼容的 JSON 数组格式。
 * 超时控制: 针对不同平台设置合理的超时时间（如 GitHub 10s，小红书 15s）。
 * 错误处理:
   * 400: URL 格式不正确。
   * 422: 页面内容为空或无法解析。
   * 500: 抓取超时或代理失效。
 * 图片处理: 转换 Markdown 时应尽可能保留图片链接。
6. 目录结构建议
/project-root
├── .env                # 环境配置文件
├── main.py             # FastAPI 入口与路由
├── crawler.py          # Crawlee 核心逻辑与 Cookie 注入
├── utils.py            # Cookie 解析与文本处理工具
└── requirements.txt    # 依赖列表
